Creating Components

The components whether atoms/molecules/organisms etc. go under the following heirarchy. The hierarchy src/components/components/* might look a bit misleading but since fractulus is a wrapper over fractal, we did not want to change something that is already working well.

The component.config.json defines the collection i.e components, specifying the component root template that needs to be used or the rendering each of the components, or the title of the collection etc. For more information about collections, read up on the fractal docs here.

fractulus
├─ *
├─ src
└─ *
   ├─ components
   └─ *
      └─ components
         ├─ test                    #The test component
         └─ component.config.json

Component Structure

Each component is a self contained entity where developers can work in isolation. Conside you having a component src/components/components/my-component. Each component folder contains the following files.

my-component
├─ my-component.config.json     # (Required) The component config file
├─ my-component.hbs             # (Required) The component template file
├─ my-component.js              # (Optional) The component stimulus script file in ES6 format
├─ my-component.scss            # (Optional) The component SASS file
├─ partials                     # (Optional) The handlebars partials you wish to include
├─ package.json                 # (Required) The package.json file which contains the rollup config for the component
└─ README.md                    # (Optional) The readme file for the component
