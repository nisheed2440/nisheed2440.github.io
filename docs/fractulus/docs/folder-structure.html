Folder Structure

Although fractal cli helps you set up the folder structure out of the box, we wanted to create a wrapper that allowed for more customizations that are project specific. Hence the following folder structure is used.

The tasks folder will be described in detail in the task customization section.

fractulus
├─ .vscode                              #The workspace setting for VSCode
│  └─ settings.json
├─ src                                  #The source files for the application
│  ├─ _fractulus                        #Internal fractulus theming files
│  │  ├─ assets                         #The fractulus theme assets 
│  │  │  ├─ favicon.ico
│  │  │  ├─ fractulus-logo-small.png
│  │  │  └─ fractulus-logo.png
│  │  ├─ scripts                        #The fractulus theme script files
│  │  │  ├─ polyfill.js                 #The es5 polyfills for IE support    
│  │  │  └─ theme.js                    
│  │  └─ styles                         #The fractulus theme style files
│  │     ├─ _variables.scss
│  │     └─ theme.scss
│  ├─ assets                            #The application assets
│  ├─ components                        #The core application stuff    
│  │  ├─ _preview                       #The fractal JS preview templates for pages and components
│  │  │  ├─ _component-preview.hbs
│  │  │  └─ _page-preview.hbs
│  │  ├─ components                     #The components for your application                     
│  │  │  ├─ test                        #Sample test components
│  │  │  └─ component.config.json       #The fractal JS components configuration file
│  │  └─ pages                          #The page templates that use your components
│  │     ├─ home.hbs                    #Sample home template
│  │     └─ pages.config.json           #The fractal JS pages configuration file
│  ├─ docs                              #The application related docs folder
│  │  └─ index.md                       #The overview document in markdown format
│  ├─ scripts                           #Application script utility files go here. These get added to app.js file
│  ├─ styles                            #Application style utility files go here. These get added to app.scss file
│  │  ├─ _common.scss
│  │  └─ _variables.scss
│  ├─ app.js                            #Application main/common JS file
│  └─ app.css                           #Application main/common CSS file
├─ tasks                                #The gulp tasks for build and deployment go here
├─ .babelrc                             #The Babel transpiler configuration
├─ .eslintrc                            #The eslint configuration
├─ .gitignore                           #Git ignore file
├─ gulpfile.js                          #The Gulpfile for build and deployment
├─ package.json                         
└─ README.md
